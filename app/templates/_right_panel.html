Rule Documentation

Overview

Rules define patterns to detect in candle data and predict the next candle's direction. All selected rules are evaluated against your candle data; each triggered rule casts a vote for bullish or bearish. The consensus determines the projected candle.

How Projection Works

  1. Select one or more rules via the checkboxes
  2. Click Project Next Candle
  3. Each rule checks its conditions against your candle data
  4. Rules that trigger vote bullish or bearish
  5. The majority vote determines the projected direction
  6. O/H/L/C values are estimated from recent candle averages

Bullish Reversal Preset (Configurable)

The Preset button creates a multi-condition rule using the v2 configurable format. All conditions must be met for the rule to trigger (AND logic).

Pattern: 2 consecutive bearish candles
Wick Comparison (upper): Candle -2 upper wick < Candle -1 upper wick (rising)
Wick Comparison (lower): Candle -2 lower wick < Candle -1 lower wick (rising)
Action: Bullish

Configurable Rule Conditions (v2)

Used by the Preset button. Conditions are AND-ed together.

Condition Type Parameters Description
pattern consecutive, direction Last N candles are all bullish/bearish
wick_comparison candle_a, candle_b, part, comparison Compare upper/lower wick size between two candles (offsets like -2, -1)
body_ratio candle, comparison, value Body-to-range ratio ≥ or ≤ threshold
volume_ratio candle, multiplier, comparison Volume vs recent average ≥ or ≤ multiplier
marubozu candle, threshold Body ≥ threshold of range (default 0.85)
doji candle, threshold Body ≤ threshold of range (default 0.10)
engulfing min_mult Current body ≥ min_mult × previous body, opposite direction

Candle Offsets

In configurable rules, candle positions use negative offsets from the end of the array: -1 = most recent candle, -2 = previous candle, -3 = two candles ago, etc.

Comparison Operators

Used by wick_comparison, body_ratio, and volume_ratio:

gt candle_b > candle_a lt candle_b < candle_a gte candle_b ≥ candle_a lte candle_b ≤ candle_a

Tips

  • Use multiple rules for better consensus — combine reversal + continuation rules
  • The Bullish Reversal preset is a good starting point for testing
  • Confidence = majority votes / total triggered rules
  • Rules are stored per-organization and persist across sessions
  • Toggle auto-refresh on Binance Live to re-project as new candles arrive
